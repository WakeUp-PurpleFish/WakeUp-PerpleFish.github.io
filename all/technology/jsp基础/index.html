jsp基础

JSP（Java Server Page）是一种创建动态页面的技术。是由静态和动态内容混合组成的动态页面（*.jsp）

静态内容：

HTML,XML,Text

动态内容：

java代码；
显示JavaBeans的属性；
使用自定义标签调用业务逻辑

JSP= HTML+JAVA

JSP的特点

内容的展现逻辑分离
使用JSP，JavaBeans and custom tags开发网页更加简明
使用组件实现软件复用（JavaBeans，custom tags）
JSP页面更新时会自动重新编译
使制作网页更容易
平台可移植
高性能，可伸缩和可靠
和企业系统应用合并后成为J2EE的一部分

Server功能强大，比较适合处理逻辑，而不善于处理显示的部分。

使用println（）函数生成HTML页面
维护HTML页面

jsp环境

​ 开发环境

​ 浏览器：Internet Explorer（IE），Netscape Communicator，Mozilla等.
​ 最简单的：像notepad，vi等基于文本的编辑器.
​ 开源集成开发环境（Open Source IDE）：Eclipse，NetBeans等.
​ 商业集成开发环境（Commercial IDE）：Websphere studio，JBuilder，Sun Java Studio等.

​ 部署和执行环境-Web容器（Web Container）
​ 自由软件：Tomcat，JBoss，Resin等。
​ 商业软件：Bea Weblogic，IBM Websphere，Oracle Application Server，Sun Java Application Server，Macromedia Jrun等.

Tomcat安装后的目录结构
部署JSP

方法一：修改Tomcat的server.xml文件
位置：Tomcat安装目录/conf/server.xml
内容：<Context path=”/jspdev”docBase=”E:/workspace/jspdev”
reloadable=”true“/>
方法二：创建jspdev.xml文件
位置：Tomcat 安装目录/conf/Catalina/localhost
内容：<Context docBase=”E:/workspace/jspdev”reloadable=”true“/>
方法三：直接将WEB工程拷贝到Tomcat的webapps目录

JSP的生命周期

编译阶段：

servlet容器编译servlet源文件，生成servlet类

三个步骤：

  • 解析JSP文件

  • 将JSP文件转为sevlet

  • 编译servlet

    初始化阶段：

加载与JSP对应的servlet类，创建其实例，并调用它的初始化方法

执行阶段：

调用与JSP对应的servlet实例的服务方法

​ 描述了JSP生命周期中一切与请求相关的交互行为，直到被销毁。 

当JSP网页完成初始化后，JSP引擎将会调用_jspService()方法。

​ _jspService()方法需要一个HttpServletRequest对象和一个HttpServletResponse对象作为它的参数。

销毁阶段：

调用与JSP对应的servlet实例的销毁方法，然后销毁servlet实例

​ JSP生命周期的销毁阶段描述了当一个JSP网页从容器中被移除时所发生的一切。
jspDestroy()方法在JSP中等价于servlet中的销毁方法。当您需要执行任何清理工作时复写jspDestroy()方法，比如释放数据库连接或者关闭文件夹等等。
